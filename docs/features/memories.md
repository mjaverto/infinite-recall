# Memories

Memories is a curated personal knowledge base built from your activity. As you work, Infinite Recall's background AI extracts durable insights — facts about how you work, notable things you encountered, and practical tips — and stores them here. You can also add entries yourself. Everything is tagged by category so you can quickly filter to what matters.

## What you see

A grouped list of memories, each displayed with its full text and one or more tag chips. Top-level tags are **Focus**, **Tips**, **System**, **Interesting**, and **Manual**. Sub-tags refine the category: Focus splits into `focused` and `distracted`; Tips breaks down into `productivity`, `health`, `communication`, and `learning`.

Above the list:

- A **search bar** for keyword lookup across all memory text.
- **Filter controls** to narrow by tag, focus sub-status, or memory source (system-generated, manually added, or AI tip).

Click any memory to open a detail sheet showing the full text, all tags, the creation date, and — when available — a link to the source conversation that triggered the extraction.

## What you can do

- Search memories by keyword.
- Filter by tag, focus status, or source type.
- Add a manual memory — your own free-text note.
- Edit or delete any memory.
- Open the source conversation linked to a memory.
- Sort and organise by date, tag, or category.
- Undo an accidental delete via the toast notification that appears immediately after.

## States

- **Loading** — spinner shown while the local store is being fetched on open.
- **Empty** — prompt shown when no memories have been created yet.
- **No results** — message shown when the active filters match nothing.
- **Local AI unavailable** — a status banner appears when the on-device LLM is offline. Existing memories remain fully usable; new auto-extractions pause until the model comes back up.
- **Conversation overlay** — a detail view slides in when you follow the source-conversation link from a memory's detail sheet.

## Behind the scenes

**Auto-extraction.** Memories are generated by three cooperating background extractors that fire on a timer and on context changes — for example, when you switch apps or finish a conversation. Each extractor sends recent screen content and transcript snippets to the local MLX Qwen text model via `LLMBridge` and writes its results into the same `memories` table, which is why the UI shows them all together:

- **System** and **Interesting** — produced by `MemoryAssistant`. System memories are durable facts about the user (preferences, recurring patterns, habits); Interesting memories are notable things pulled from screen content or transcripts.
- **Tip** — produced by `InsightAssistant`. Tips are actionable advice, tagged by domain: productivity, health, communication, or learning.
- **Focus** (`focused` / `distracted`) — produced by `FocusAssistant` from the active-app and recent-activity signal.

Confidence scores are applied after inference; extractions that fall below the threshold are discarded before anything is written to storage.

**Deduplication.** Before persisting a new memory, the assistant compares it against the most recent ~20 stored memories. Candidates that express the same fact as an existing entry — even if phrased differently — are dropped.

**Storage.** Memories live in the local GRDB `memories` table. Each row records the screenshot ID and source app that triggered the extraction, which is what backs the "open source conversation" link in the detail view.

**Manual memories.** Memories you write yourself are tagged `Manual` and are otherwise identical to auto-extracted ones — fully searchable, filterable, and editable.

**Gating.** Auto-extraction runs whenever the assistant is enabled and the per-frame extraction interval (default ~10 minutes) has elapsed — it does **not** require the Mac to be idle. Memory extraction is in the lightweight tier alongside transcription and OCR, so it runs while you are actively using the machine. It does pause on battery / Low Power Mode along with the rest of the AC-only pipeline; only the heavier conversation-summarisation and knowledge-graph passes wait for idle as well. The full gate model is in the Activity doc. The Memories UI itself remains fully functional regardless of capture state — only new background extractions pause.
